# 🚀 Развёртывание SaveMOD 24/7 бесплатно (без карты)

Боту нужно работать **круглосуточно**, чтобы ловить удаления/редактирования
сообщений в реальном времени. Ниже — бесплатные способы без привязки карты.

Бот работает в режиме **long-polling** (сам опрашивает Telegram), поэтому вебхук
настраивать НЕ нужно. Внутри есть маленький health-сервер (`/` и `/health`) —
он нужен только для того, чтобы бесплатные хостинги не «усыпляли» сервис.

---

## ✅ Вариант 1 — Koyeb (рекомендую: карта обычно не нужна, быстрый)

Koyeb даёт 1 бесплатный веб-сервис (512 МБ RAM). Карту при регистрации
обычно **не спрашивают** (могут попросить, только если не смогут автоматически
подтвердить, что вы человек — тогда см. Вариант 2 без карты).

1. Зайдите на **https://www.koyeb.com** → **Sign up** через GitHub-аккаунт.
2. **Create Web Service** → **GitHub** → выберите репозиторий
   `tt2819ais-arch/SaveMode`, ветка `main`.
3. Билд: **Dockerfile** (Koyeb определит его автоматически).
4. Instance: выберите **Free** (512 MB, регион Frankfurt).
5. **Environment variables** — добавьте:
   - `BOT_TOKEN` = ваш токен от @BotFather
   - `OWNER_ID` = `5426581017`
6. **Ports**: порт `8000`, тип `HTTP` (обычно определяется сам).
7. **Deploy**. Через пару минут получите адрес вида
   `https://<имя>-<орг>.koyeb.app`.
8. **Keep-alive** (чтобы не засыпал через 1 час без трафика): см. раздел
   «UptimeRobot» ниже — пингуйте этот адрес каждые 5 минут.

---

## ✅ Вариант 2 — Render (карта точно НЕ нужна)

Render **гарантированно не требует карту** при регистрации. Минус: бесплатный
сервис засыпает через 15 мин без трафика (лечится keep-alive пингом) и есть
лимит 750 часов/мес (при работе 24/7 это ~744 ч — как раз укладывается).

1. **https://render.com** → **Sign up** (можно через GitHub, без карты).
2. **New +** → **Web Service** → подключите репозиторий `SaveMode`.
3. **Runtime: Docker** (Render увидит Dockerfile).
4. Instance Type: **Free**.
5. **Environment** → добавьте `BOT_TOKEN` и `OWNER_ID` (как выше).
6. **Create Web Service**. Получите адрес `https://<имя>.onrender.com`.
7. Обязательно настройте **keep-alive** (иначе уснёт через 15 мин).

## 🔔 Keep-alive через UptimeRobot (бесплатно, без карты)

Держит сервис «тёплым» 24/7, пингуя его каждые 5 минут.

1. **https://uptimerobot.com** → регистрация (карта не нужна, 50 мониторов).
2. **+ New monitor** → тип **HTTP(s)**.
3. URL: адрес вашего сервиса + `/health`
   (напр. `https://<имя>.koyeb.app/health` или `https://<имя>.onrender.com/health`).
4. Monitoring interval: **5 минут**. Сохраните.

Готово — бот не будет засыпать.

Если облако капризничает — запуск на своём Android через **Termux** самый

## 💾 ВАЖНО: сохраняйте базу между редеплоями

SQLite-база (`savemod.db`) и кэш медиа лежат в файловой системе контейнера.
На бесплатных хостах (Koyeb/Render) она **эфемерная** — стирается при каждом
редеплое/рестарте. Последствия:

- теряется история сохранённых сообщений (нечего восстанавливать, если удалят
  сообщение до того, как оно снова сохранится);
- теряется запись Business-подключения (Telegram присылает `business_connection`
  только один раз при подключении и **не повторяет** при рестарте).

Бот умеет **самовосстанавливать** запись подключения через
`get_business_connection`, а владелец берётся из `OWNER_ID` как фолбэк — поэтому
`.команды` и уведомления об удалении/правках работают даже после сброса базы.
Но чтобы **не терять историю сообщений**, вынесите базу на постоянное хранилище:

```
DB_PATH=/data/savemod.db   # путь на смонтированном постоянном томе (volume)
```

- **Koyeb / Render** — подключите Persistent Volume и смонтируйте, например, в
  `/data`, затем задайте `DB_PATH=/data/savemod.db`.
- **Termux/свой ПК** — база и так лежит на постоянном диске, ничего делать не
  нужно.

Всегда указывайте **`OWNER_ID`** (ваш Telegram id) — это гарантирует работу
`.команд` и уведомлений сразу после старта, даже с пустой базой.

## ⚠️ Что НЕ подходит для этого бота

- **PythonAnywhere free** — «always-on task» только на платном тарифе; на
  бесплатном непрерывный бот работать не будет.
- **Replit / Glitch** — бесплатные реплы засыпают, «Always On» теперь платный.
- **Railway** — бесплатный триал ограничен и в итоге требует оплату.
- **fly.io** — требует привязку карты при регистрации.
