# SaveMOD 🤖

Telegram-бот для **Telegram Business Mode**, который сохраняет удалённые сообщения,
отслеживает правки, предупреждает о мошенниках и добавляет полезные команды и игры
в ваши чаты.

> ⚠️ **Требуется Telegram Premium** у владельца (обязательное условие Business Mode).

---

## 🚀 Возможности

- 🗑 **Сохранение удалённых сообщений** — когда собеседник удаляет сообщение, вы
  получаете его копию в личку.
- ✏️ **Отслеживание правок** — видите, что было и что стало (с blockquote).
- 🚨 **Антискам** — real-time предупреждение о подозрительных сообщениях/ботах
  (best-effort эвристика, не 100% защита).
- 📋 **Меню команд** с инлайн-кнопками и зацикленной навигацией.
- 🎮 **Игры на двоих** через инлайн-кнопки: крестики-нолики, дуэль, кубик,
  монетка, «закрась поле», Wordle (ставки — игровые очки ⭐, НЕ реальные Telegram Stars).
- 🛠 **Админ-панель** для владельца: список пользователей, статистика, рассылка,
  личные сообщения.

## 📋 Команды

**Работа с сообщениями:** `.help` `.kawaii` `.love` `.sw` `.type` `.check`
**Инструменты:** `.qr` `.tr` `.calc` `.pass` `.mock` `.rev` `.roll` `.pick` `.count` `.b64` `.spoiler`
**Игры:** `.ttt` `.duel` `.dice` `.flip` `.bw` `.wordle`
**Обработка:** `.afk` `.info` `.nk` `.short` `.fv` `.gifts` `.status` `.story` `.time` `.yars`
**Остальное:** `.gosu` `.lq`

Служебное: `.autodel on|off` — авто-удаление сообщений-команд после отправки
(по умолчанию включено; команды, которые заменяют ваше сообщение результатом,
не удаляются).

### Меню и цветные кнопки

Меню — плоский грид всех команд (как в референсе): нажми на кнопку, чтобы
увидеть описание команды. Активная команда красится в **зелёный** «сюда
нажимать», цвет динамически переезжает через `edit_message_reply_markup`.

> **Про цвет кнопок.** Telegram Bot API поддерживает поле
> `InlineKeyboardButton.style` со значениями **`success` (зелёная)**,
> **`primary` (синяя)**, **`danger` (красная)**. Это настоящий цвет фона
> кнопки, без Mini App / WebApp. Используем с умом (один зелёный акцент вместо
> «радуги»): зелёный — целевое действие/«сюда нажимать» и подтверждения,
> синий — навигация (меню/назад/разделы), красный — отмена/отключение/сдаться.
> Опционально доступен `icon_custom_emoji_id` (кастомное/анимированное эмодзи
> на кнопке) — работает у ботов с купленным на Fragment юзернеймом либо в
> сообщениях бота, если у владельца есть Telegram Premium.

Команды пишутся **вами** (владельцем) в подключённых чатах, начиная с точки.

### 🟩 Wordle (`.wordle`)

1. Наберите `.wordle` в чате — появится кнопка **«🔐 Загадать слово»**
   (deep-link `t.me/<бот>?start=wordle_<id>`).
2. Нажмите её — откроется личка с ботом, там загадайте секретное слово
   из 5 букв (русское **или** английское). Так слово скрыто от соперника.
3. Игра активируется в исходном чате; соперник угадывает слово, просто
   набирая слова из 5 букв. Обратная связь по буквам:
   🟩 буква на месте · 🟨 буква есть, но не там · ⬜ буквы нет.

В личке с ботом (тест-режим) можно играть в одиночку — загадать и угадывать самому.

> **Про цвет доски.** Клетки раскрашены **самой кнопкой** через
> `InlineKeyboardButton.style`, а не эмодзи-квадратом в тексте:
> буква на месте → **зелёная** кнопка (текст = буква), буквы нет → **серая**
> кнопка (текст = буква). Жёлтого `style` в API нет, поэтому «буква есть, но не
> на месте» — единственный оставшийся эмодзи-фолбэк: 🟨 рядом с буквой. Кнопка
> «Сдаться» — красная. Цвета видны на свежем клиенте Telegram.

## 🛠 Админ-панель (только владелец)

- `/admin` — открыть панель
- 👥 Список подключённых пользователей (имя, @username, id, дата, статус)
- 📊 Статистика (всего/активных подключений, сохранённых сообщений, игр)
- 📢 Рассылка всем (с подтверждением и отчётом доставлено/не доставлено)
- ✉️ Сообщение конкретному пользователю (`id | текст` или `@username | текст`)

Все админ-команды строго ограничены `OWNER_ID`.

## ⚙️ Установка и запуск

### 1. Включите Business Mode у бота (ОБЯЗАТЕЛЬНО!)

В [@BotFather](https://t.me/BotFather):
```
/mybots → выберите бота → Bot Settings → Business Mode → Turn on
```
Без этого бот **не сможет** подключиться к вашему аккаунту.

### 2. Конфигурация

Скопируйте `.env.example` в `.env` и заполните:
```
BOT_TOKEN=токен_от_BotFather
OWNER_ID=ваш_telegram_id
# Опционально — для транскрибации голосовых в .short (Whisper API):
# OPENAI_API_KEY=sk-...
```

**Опциональные возможности:**
- `.fv` (смена голоса) требует **ffmpeg** — в Docker-образе он уже есть;
  в Termux: `pkg install ffmpeg`.
- `.short` расшифровывает голосовые, только если задан `OPENAI_API_KEY`
  (используется Whisper API). Без ключа `.short` делает пересказ текста
  (работает всегда) и показывает длительность голосового.

### 3. Запуск через Docker (рекомендуется для 24/7)

```bash
docker build -t savemod .
docker run -d --name savemod --restart unless-stopped \
  --env-file .env \
  -v $(pwd)/data:/app/data \
  savemod
```

### 4. Запуск без Docker

```bash
pip install -r requirements.txt
python run.py
```

### 5. Подключите бота в Telegram

Настройки → **Telegram Business** → **Чат-боты** → добавьте `@ваш_бот`.

После подключения вы получите приветственное сообщение с кнопкой
«Команды и функционал».

## 🌐 Хостинг 24/7

Бот использует **long-polling** и должен работать постоянно, чтобы ловить
события удаления сообщений в реальном времени. GitHub Actions cron **не подходит**
(не 24/7 → пропустит удаления).

📖 **Подробная пошаговая инструкция по бесплатному деплою без карты — в
[DEPLOY.md](DEPLOY.md).**

Бесплатные варианты без привязки карты (кратко):
- **Koyeb** — карту обычно не спрашивают; + keep-alive пинг (UptimeRobot).
- **Render** — карта точно не нужна; + keep-alive пинг (засыпает через 15 мин).
- **Termux** на своём Android — самый надёжный, нужно только своё устройство.

Для keep-alive у бота есть встроенный health-сервер (`/` и `/health`) на порту
`PORT` (по умолчанию 8000) — пингуйте его каждые 5 минут через бесплатный
UptimeRobot, чтобы хостинг не «усыплял» сервис. Вебхук настраивать не нужно —
бот сам опрашивает Telegram (long-polling).

## 🔒 Безопасность

- Токен бота хранится **только** в переменных окружения, не в коде и не в репозитории.
- Данные пользователей — локально в SQLite, третьим лицам не передаются.
- Игровые очки ⭐ — **симулированные**, не реальные Telegram Stars.
- Антискам — best-effort, не гарантирует 100% защиту.

## 🧱 Структура проекта

```
run.py                 — точка входа (long-polling + health-сервер)
bot/
  config.py            — конфигурация из ENV
  storage.py           — SQLite CRUD
  keepalive.py         — health-сервер для keep-alive (Koyeb/Render)
  handlers/
    business.py        — подключение, удаление, правки, роутинг
    commands.py        — все .команды
    games.py           — логика игр
    callbacks.py       — инлайн-кнопки (меню, игры, админ)
    admin.py           — админ-панель, /start, /admin
    antiscam.py        — антискам-анализ
  utils/
    constants.py       — тексты, описания, паттерны
    keyboards.py       — инлайн-клавиатуры
    text_tools.py      — sw, kawaii, love, форматирование, пересказ
    audio.py           — обработка голоса через ffmpeg (.fv)
    stt.py             — опц. транскрибация (Whisper) для .short
tests/
  test_logic.py        — юнит-тесты логики
```
